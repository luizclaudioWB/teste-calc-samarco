## PROMPT PARA MANUS IA — Frontend "Motor de Cálculo Energético SAMARCO"

### CONTEXTO DO PROJETO

Preciso criar um frontend completo em **React + TypeScript + Vite** para o "Motor de Cálculo Energético" da SAMARCO. O backend já existe em Java (Quarkus) e expõe uma API **GraphQL** no endpoint `http://localhost:8080/graphql`. O sistema reproduz cálculos de planejamento energético que antes eram feitos em Excel (~R$ 392M/ano).

O frontend deve:
1. Visualizar todos os resultados dos **10 cálculos encadeados** (Abas 4, 6, 7, 9, 12, 13, 14, 15, 16)
2. Permitir validação comparando resultado do frontend vs API (painel verde/vermelho)
3. Ser moderno, responsivo e profissional — dark theme premium

Os arquivos anexos são:
- **SAMARCO-PLANILHA-REPLY.xlsx** — planilha Excel original com todas as fórmulas (fonte de verdade)
- **jsonParaComparacao.json** — retorno real da API GraphQL (JSON com todos os dados calculados)

### STACK TECNOLÓGICA OBRIGATÓRIA

- **React 18+** com **TypeScript**
- **Vite** como bundler
- **React Router v6** para roteamento SPA
- **Apollo Client** para comunicação GraphQL
- **Ant Design (antd) 5+** — tema dark customizado
- **Recharts** ou **Apache ECharts** para gráficos
- **Montserrat** como fonte principal (Google Fonts)
- **CSS Modules** ou **Styled Components**

### API GRAPHQL — QUERY COMPLETA

O backend expõe UMA query que retorna tudo:

```graphql
query {
  calcularMotorCompleto {
    # Cálculo 01 — Produção (tms)
    producaoTms { area mes valor }

    # Cálculo 03 — Consumo Área (MWh)
    consumoAreaTotalMWh { mes valor }
    consumoMG_MWh { mes valor }
    consumoES_MWh { mes valor }
    percentualMG { mes valor }
    percentualES { mes valor }

    # Cálculo 04 — Geração Própria
    geracaoGuilmanMWh { mes valor }
```

### OBSERVAÇÕES IMPORTANTES

1. **Uma única query GraphQL** retorna TODOS os dados — faça a chamada uma vez e distribua para as páginas via Context ou state management
2. O arquivo `jsonParaComparacao.json` anexo tem o retorno real da API — use como mock enquanto desenvolve sem o backend rodando
3. **Tabelas são o core** — devem ser bonitas e fáceis de ler com muitos números. Inspiração: planilha Excel mas com visual premium
4. **Cadeia de dependência** é importante visualmente — o usuário precisa entender que os 10 cálculos se encadeiam na ordem 01→02→03→04→05→13→14→15→16
5. Valores devem bater 100% com o Excel — a página de Validação é a prova disso
6. O backend roda em `http://localhost:8080` — configurar o Apollo Client com essa URL
7. **Cálculo 16 (Resumo Geral) é a página mais importante** — é a consolidação final que os gestores vão usar para tomada de decisão. Dê atenção especial ao design desta página
8. As 5 classes de custo (50610002-50610008) são o eixo central da visualização financeira
